# 用户角色与权限设计

***

## 4.1 角色定义

### 1）系统管理员

权限最大，可维护所有基础数据、账户、抽取规则、日志、必抽设置。

### 2）业务管理员

负责单位、人员、指挥组、任务方案录入与维护；可发起抽考，但不能管理系统参数。

### 3）抽考操作员

仅能按已设规则执行抽考，不能修改基础数据和必抽设置。

### 4）查看用户

仅查看结果和导出打印。

## 五、核心功能详细设计

### 5.1 抽考算法设计

#### 基本原则

* 公平随机

* 满足必抽约束

* 满足去重规则

* 结果可重现（可选：记录随机种子）

***

#### 指挥组抽取算法

##### 输入

* 候选指挥组池

* 抽取数量N

* 必抽指挥组列表

* 排除列表

* 其他约束

##### 逻辑步骤

1. 过滤停用、不可抽、排除项

2. 读取必抽列表

3. 若必抽数量 > N，则提示规则冲突

4. 将必抽对象先放入结果集

5. 从剩余候选池中随机补足到N

6. 按最终顺序生成结果表

```
候选池 = 所有可抽指挥组 - 排除对象
必抽池 = 候选池中标记为本次必抽的对象

如果 必抽池数量 > N:
    返回错误：必抽数量超过抽取数量

结果集 = 必抽池

剩余池 = 候选池 - 必抽池
剩余数量 = N - 结果集数量

从剩余池中随机抽取 剩余数量 条加入结果集
如设置固定位置，则按位置重排
输出结果集
```



#### 最终配对算法

##### 默认规则

按抽取序号一一对应：

* 任务1 -> 指挥组1

* 任务2 -> 指挥组2

##### 异常处理

* 如果任务数量大于指挥组数量，则后续任务可挂到同一组，或提示配置错误

* 如果指挥组数量大于任务数量，则可一组多任务或按规则补空

##### 推荐限制

为了简洁，建议系统默认要求：

* 抽取指挥组数量 = 抽取任务数量
  这样结果表最清晰。
